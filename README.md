# Движок регулярных выражений

Реализация `regex_matcher`, повторяющая поведение эталона из `example/`:
для каждой строки текста и каждой регулярки печатается число непересекающихся
совпадений (`len(re.findall(...))`), либо `-1` для некорректной регулярки.

## Подход

- разбор регулярки рекурсивным спуском в дерево (`|`, `*`, `()`, конкатенация, `\`);
- матчинг — рекурсивный бэктрекинг: узел пытается совпасть, а продолжение
  проверяет остаток шаблона; первый успех при жадной звезде и порядке
  альтернатив даёт тот же выбор, что и Python `re`;
- `findall` — непересекающиеся совпадения слева направо, с обработкой пустых;
- глубина рекурсии может достигать длины строки, поэтому матчинг считается
  в потоке с большим стеком.

## Сборка

```
bash package.sh
```

Собирает `regex_matcher` и упаковывает его в `test_bins.tar.gz`.

## Запуск

```
./regex_matcher <re_file> <text_file> <out_file>
```
